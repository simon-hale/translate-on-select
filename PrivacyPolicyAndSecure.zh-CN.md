# Translate-on-Select 隐私权声明与安全处理说明

更新日期：2026-03-17

本声明适用于 `Translate-on-Select` Chrome 扩展的当前实现版本。

本扩展的单一用途是：将用户在网页上主动选中的文本翻译为用户选择的目标语言。为实现这一功能，本扩展会处理部分与网页内容、翻译请求和本地设置有关的数据。根据 Chrome Web Store 关于用户数据、隐私政策和安全处理的要求，本声明说明本扩展会处理哪些数据、如何使用这些数据、会将数据分享给谁、保存多久，以及用户可以如何控制和删除这些数据。

## 1. 我们会处理哪些数据

本扩展当前会处理以下数据：

- 用户在网页中主动选中的文本。
- 用户选择的目标语言。
- 用户在弹窗或设置页中填写或选择的配置项，包括：
  - 工作模式（自定义服务器模式 / 自定义 API 模式）
  - API 提供方选择
  - 自定义服务器 URL
  - 服务器端点类型
  - HTTP 方法
  - DeepL API Key
  - DeepSeek API Key
  - DeepL Endpoint 类型
  - DeepSeek 流式输出开关
  - 界面语言
  - 主题模式
- 翻译返回结果。
- 用户点击“复制”时，被复制到本机剪贴板的翻译结果。

本扩展不提供用户注册、登录、广告、统计分析、营销邮件或画像功能。根据当前代码实现，本扩展不会主动收集或上传以下信息：

- 用户姓名、邮箱、电话、住址等身份信息。
- 用户的完整浏览历史。
- 用户的 Cookie、表单内容、书签、下载记录或支付信息。
- 未经用户触发的完整网页正文。

需要说明的是：本扩展会在 `<all_urls>` 匹配的网页中注入内容脚本，以便在用户选中文本时显示翻译按钮。虽然它运行在网页环境中，但当前实现只在用户选中文本并触发翻译时读取所选文本，不会主动上传整个页面内容。

## 2. 我们如何收集这些数据

本扩展通过以下方式处理数据：

- 当用户访问网页时，内容脚本会在页面中运行，以支持“划词后显示翻译按钮”的功能。
- 当用户主动选中文本并点击翻译按钮时，扩展会读取该段选中文本，并将其发送给后台脚本处理。
- 当用户在弹窗或设置页中填写 API Key、服务器地址或其他配置时，扩展会把这些配置保存在当前浏览器的 `chrome.storage.local` 中。
- 当用户点击“复制”按钮时，翻译结果会被写入本机剪贴板。

本扩展不会在用户没有进行翻译操作时，自动持续上传用户选择的网页文本。

## 3. 我们如何使用这些数据

本扩展仅将处理到的数据用于以下用途：

- 执行用户请求的翻译功能。
- 根据用户设置选择翻译目标语言、翻译服务或服务器端点。
- 在页面中显示翻译结果。
- 保存用户的本地偏好设置和连接配置，避免重复输入。
- 在用户点击“复制”时，把翻译结果复制到本机剪贴板。

除实现上述翻译功能和必要设置保存外，本扩展不会将用户数据用于以下用途：

- 个性化广告。
- 用户画像或行为分析。
- 向数据经纪商、广告平台或信息转售方出售或转让数据。
- 信用评估、放贷或其他与翻译功能无关的目的。

## 4. 我们会把数据分享给谁

本扩展是否会把数据发送到第三方，取决于用户当前选择的翻译模式。

### 4.1 直连 API 模式

如果用户选择直连 API 模式，本扩展会把翻译所需的数据直接发送给用户选择的第三方翻译服务提供方：

- `DeepL`
- `DeepSeek`

在这种模式下：

- 用户选中的文本会被发送给所选提供方，以完成翻译。
- 目标语言参数会一并发送。
- 对应的 API Key 会作为认证信息发送给所选提供方。

### 4.2 自定义服务器模式

如果用户选择自定义服务器模式，本扩展会把翻译所需的数据发送到用户自己填写的服务器地址。

在当前实现中，发送给自定义服务器的数据包括：

- 用户选中的文本
- 目标语言

在当前实现中，本扩展不会把 `DeepL API Key` 或 `DeepSeek API Key` 一并发送给自定义服务器；这些 Key 只用于直连 API 模式。

请注意：

- 自定义服务器由用户自行配置，其隐私与安全实践取决于该服务器的运营者。
- 如果用户把数据发送到自己控制或信任的服务器，应由该服务器运营者负责提供相应的隐私声明和安全保护。

### 4.3 开发者自身

根据当前代码实现，本扩展没有内置开发者自有的统计、埋点、广告或遥测上报接口，也没有把用户选中文本或 API Key 固定发送到开发者服务器的逻辑。

但如果用户自行配置了某个自定义服务器，而该服务器由开发者本人运营，那么用户数据将会按“自定义服务器模式”被发送到该服务器。

## 5. 本地存储、保留期限与删除方式

本扩展会将以下配置保存在当前浏览器的 `chrome.storage.local` 中：

- `backendMode`
- `apiBrand`
- `serverUrl`
- `httpMethod`
- `apiSelectServer`
- `deeplApiKey`
- `deepseekApiKey`
- `deeplEndpoint`
- `targetLanguage`
- `streamDeepseek`
- `uiLanguage`
- `themeMode`

保留期限如下：

- 上述配置会一直保留在当前浏览器中，直到用户主动清除、重置扩展配置、清除浏览器中的扩展数据，或卸载本扩展。
- 用户选中的文本和翻译结果不会被本扩展长期持久化保存；它们主要只在当前请求处理过程和当前页面显示期间保留在内存中。

用户可以通过以下方式管理或删除数据：

- 在设置页中清除服务器配置或 API Key。
- 在浏览器扩展管理页面中删除扩展数据。
- 直接卸载本扩展。
- 不再使用翻译功能，或避免选择包含敏感信息的文本。

## 6. 安全处理说明

我们会尽量按照 Chrome Web Store 的用户数据安全处理要求来说明当前实现：

- 在直连 API 模式下，当前代码使用的是 `https://api.deepl.com`、`https://api-free.deepl.com` 和 `https://api.deepseek.com` 等 HTTPS 接口发送翻译请求。
- 在自定义服务器模式下，服务器地址由用户自行填写。为了保护选中文本等数据的传输安全，用户应仅配置受信任且启用了 HTTPS 的服务器地址。
- 如果用户填写的是非 HTTPS 地址，则数据传输可能不具备加密保护；这种风险由该自定义服务器配置带来。
- API Key 属于认证信息。当前实现会把 API Key 保存在 `chrome.storage.local` 中，便于后续复用；用户可以随时在设置页中删除它们。
- 当前实现不会公开展示用户的 API Key，也不会把支付信息作为扩展功能的一部分进行收集。

同时也请用户注意：

- 不要使用本扩展翻译密码、访问令牌、验证码、Cookie、支付信息、身份证号、医疗信息或其他高度敏感内容，除非你明确知晓并接受所选翻译服务或自定义服务器的处理方式。
- 如果你使用第三方翻译服务或自定义服务器，这些服务的日志、保留策略和进一步处理规则由对应服务提供方决定，不受本扩展直接控制。

## 7. 第三方服务说明

当用户启用相应模式时，以下第三方可能会接收翻译请求数据：

- DeepL
- DeepSeek
- 用户自行配置的自定义服务器运营方

这些第三方对数据的进一步处理、日志保留、合规义务和跨境传输安排，适用它们各自的条款、隐私政策和安全措施。本扩展无法替代这些第三方作出承诺。

## 8. 与 Chrome Web Store 单一用途和最小使用原则的一致性说明

本扩展对用户数据的使用限于其单一用途：对用户主动选中的网页文本进行翻译并显示翻译结果。

根据当前代码实现，本扩展不会：

- 将用户数据用于广告或营销。
- 将用户数据出售给任何第三方。
- 将用户数据提供给数据经纪商、广告平台或信息转售方。
- 将用户数据用于信用评估、放贷决策或其他与翻译功能无关的用途。

## 9. 政策更新

如果本扩展后续新增了新的数据处理方式、第三方服务、同步能力、账户系统、分析统计或其他会影响隐私与安全的功能，本声明会相应更新，并以更新后的版本为准。
