# Translate-on-Select 隱私權聲明與安全處理說明

更新日期：2026-03-17

語言版本：[English](./PrivacyPolicyAndSecure.md) | [簡體中文](./PrivacyPolicyAndSecure.zh-CN.md) | **繁體中文**

本聲明適用於 `Translate-on-Select` Chrome 擴充功能的目前實作版本。

本擴充功能的單一用途是：將使用者在網頁上主動選取的文字翻譯為使用者所選的目標語言。為了實現這項功能，本擴充功能會處理部分與網頁內容、翻譯請求以及本機設定有關的資料。根據 Chrome Web Store 關於使用者資料、隱私政策與安全處理的要求，本聲明說明本擴充功能會處理哪些資料、如何使用這些資料、會將資料分享給誰、保存多久，以及使用者可以如何控制與刪除這些資料。

## 1. 我們會處理哪些資料

本擴充功能目前會處理以下資料：

- 使用者在網頁中主動選取的文字。
- 使用者選擇的目標語言。
- 使用者在彈出視窗或設定頁面中填寫或選擇的設定項目，包括：
  - 運作模式（自訂伺服器模式 / 自訂 API 模式）
  - API 服務提供者選擇
  - 自訂伺服器 URL
  - 伺服器端點類型
  - HTTP 方法
  - DeepL API Key
  - DeepSeek API Key
  - DeepL Endpoint 類型
  - DeepSeek 串流輸出開關
  - 介面語言
  - 主題模式
- 翻譯返回結果。
- 使用者點擊「複製」時，被複製到本機剪貼簿的翻譯結果。

本擴充功能不提供使用者註冊、登入、廣告、統計分析、行銷郵件或使用者画像功能。根據目前程式碼實作，本擴充功能不會主動收集或上傳以下資訊：

- 使用者姓名、電子郵件、電話、住址等身分資訊。
- 使用者的完整瀏覽紀錄。
- 使用者的 Cookie、表單內容、書籤、下載紀錄或付款資訊。
- 未經使用者觸發的完整網頁正文內容。

需要說明的是：本擴充功能會在 `<all_urls>` 匹配的網頁中注入內容指令碼，以便在使用者選取文字時顯示翻譯按鈕。雖然它執行於網頁環境中，但目前實作只會在使用者選取文字並觸發翻譯時讀取所選文字，不會主動上傳整個頁面內容。

## 2. 我們如何收集這些資料

本擴充功能會透過以下方式處理資料：

- 當使用者造訪網頁時，內容指令碼會在頁面中執行，以支援「劃詞後顯示翻譯按鈕」的功能。
- 當使用者主動選取文字並點擊翻譯按鈕時，擴充功能會讀取該段選取文字，並將其傳送給背景指令碼處理。
- 當使用者在彈出視窗或設定頁面中填寫 API Key、伺服器位址或其他設定時，擴充功能會將這些設定保存在目前瀏覽器的 `chrome.storage.local` 中。
- 當使用者點擊「複製」按鈕時，翻譯結果會被寫入本機剪貼簿。

本擴充功能不會在使用者沒有進行翻譯操作時，自動持續上傳使用者所選的網頁文字。

## 3. 我們如何使用這些資料

本擴充功能僅將處理到的資料用於以下用途：

- 執行使用者要求的翻譯功能。
- 根據使用者設定選擇翻譯目標語言、翻譯服務或伺服器端點。
- 在頁面中顯示翻譯結果。
- 保存使用者的本機偏好設定與連線設定，避免重複輸入。
- 在使用者點擊「複製」時，將翻譯結果複製到本機剪貼簿。

除實現上述翻譯功能與必要的設定保存外，本擴充功能不會將使用者資料用於以下用途：

- 個人化廣告。
- 使用者画像或行為分析。
- 向資料經紀商、廣告平台或資料轉售方出售或轉讓資料。
- 信用評估、放貸決策或其他與翻譯功能無關的用途。

## 4. 我們會將資料分享給誰

本擴充功能是否會將資料傳送給第三方，取決於使用者目前選擇的翻譯模式。

### 4.1 直連 API 模式

如果使用者選擇直連 API 模式，本擴充功能會將翻譯所需的資料直接傳送給使用者所選擇的第三方翻譯服務提供者：

- `DeepL`
- `DeepSeek`

在這種模式下：

- 使用者選取的文字會被傳送給所選提供者，以完成翻譯。
- 目標語言參數會一併傳送。
- 對應的 API Key 會作為驗證資訊傳送給所選提供者。

### 4.2 自訂伺服器模式

如果使用者選擇自訂伺服器模式，本擴充功能會將翻譯所需的資料傳送到使用者自行填寫的伺服器位址。

在目前實作中，傳送給自訂伺服器的資料包括：

- 使用者選取的文字
- 目標語言

在目前實作中，本擴充功能不會將 `DeepL API Key` 或 `DeepSeek API Key` 一併傳送給自訂伺服器；這些 Key 僅用於直連 API 模式。

請注意：

- 自訂伺服器由使用者自行設定，其隱私與安全實務取決於該伺服器的營運者。
- 如果使用者將資料傳送到自己控制或信任的伺服器，則應由該伺服器營運者負責提供相應的隱私聲明與安全保護。

### 4.3 開發者自身

根據目前程式碼實作，本擴充功能沒有內建開發者自有的統計、埋點、廣告或遙測上報介面，也沒有將使用者選取文字或 API Key 固定傳送到開發者伺服器的邏輯。

但如果使用者自行設定了某個自訂伺服器，而該伺服器由開發者本人營運，那麼使用者資料將會依照「自訂伺服器模式」被傳送到該伺服器。

## 5. 本機儲存、保存期限與刪除方式

本擴充功能會將以下設定保存在目前瀏覽器的 `chrome.storage.local` 中：

- `backendMode`
- `apiBrand`
- `serverUrl`
- `httpMethod`
- `apiSelectServer`
- `deeplApiKey`
- `deepseekApiKey`
- `deeplEndpoint`
- `targetLanguage`
- `streamDeepseek`
- `uiLanguage`
- `themeMode`

保存期限如下：

- 上述設定會一直保留在目前瀏覽器中，直到使用者主動清除、重設擴充功能設定、清除瀏覽器中的擴充資料，或解除安裝本擴充功能。
- 使用者選取的文字與翻譯結果不會被本擴充功能長期持久化保存；它們主要只會在目前請求處理期間，以及目前頁面顯示期間保留在記憶體中。

使用者可以透過以下方式管理或刪除資料：

- 在設定頁面中清除伺服器設定或 API Key。
- 在瀏覽器擴充功能管理介面中刪除擴充資料。
- 直接解除安裝本擴充功能。
- 不再使用翻譯功能，或避免選取包含敏感資訊的文字。

## 6. 安全處理說明

我們會盡量依照 Chrome Web Store 對使用者資料安全處理的要求來說明目前實作：

- 在直連 API 模式下，目前程式碼使用的是 `https://api.deepl.com`、`https://api-free.deepl.com` 和 `https://api.deepseek.com` 等 HTTPS 介面傳送翻譯請求。
- 在自訂伺服器模式下，伺服器位址由使用者自行填寫。為了保護所選文字等資料在傳輸過程中的安全，使用者應僅設定受信任且已啟用 HTTPS 的伺服器位址。
- 如果使用者填寫的是非 HTTPS 位址，則資料傳輸可能不具備加密保護；這項風險來自該自訂伺服器設定本身。
- API Key 屬於驗證資訊。目前實作會將 API Key 保存在 `chrome.storage.local` 中，以便後續重複使用；使用者可以隨時在設定頁面中將其刪除。
- 目前實作不會公開顯示使用者的 API Key，也不會將付款資訊作為擴充功能功能的一部分加以收集。

同時也請使用者注意：

- 不要使用本擴充功能翻譯密碼、存取權杖、驗證碼、Cookie、付款資訊、身分證號、醫療資訊或其他高度敏感內容，除非你已明確了解並接受所選翻譯服務或自訂伺服器對這些資料的處理方式。
- 如果你使用第三方翻譯服務或自訂伺服器，這些服務的日誌、保存政策以及進一步處理規則由對應服務提供者決定，並不受本擴充功能直接控制。

## 7. 第三方服務說明

當使用者啟用相應模式時，以下第三方可能會接收翻譯請求資料：

- DeepL
- DeepSeek
- 使用者自行設定的自訂伺服器營運方

這些第三方對資料的進一步處理、日誌保存、合規義務以及跨境傳輸安排，適用其各自的條款、隱私政策與安全措施。本擴充功能無法替代這些第三方作出承諾。

## 8. 與 Chrome Web Store 單一用途與最小使用原則的一致性說明

本擴充功能對使用者資料的使用僅限於其單一用途：對使用者主動選取的網頁文字進行翻譯，並顯示翻譯結果。

根據目前程式碼實作，本擴充功能不會：

- 將使用者資料用於廣告或行銷。
- 將使用者資料出售給任何第三方。
- 將使用者資料提供給資料經紀商、廣告平台或資料轉售方。
- 將使用者資料用於信用評估、放貸決策或其他與翻譯功能無關的用途。

## 9. 政策更新

如果本擴充功能後續新增了新的資料處理方式、第三方服務、同步能力、帳戶系統、分析統計或其他會影響隱私與安全的功能，本聲明會相應更新，並以更新後的版本為準。
